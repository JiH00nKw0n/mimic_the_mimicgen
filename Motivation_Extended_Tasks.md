# Bias in Synthetic Data Generation - Motivation 실험 (2): Task 확장

## Recall

### Backgrounds

- **모방학습에는 다양한 조건을 담은 큰 데모 데이터셋이 필요하다** — 물체가 어디에 놓이느냐에 따라 필요한 동작이 달라지므로, 소수의 사람 시연만으로는 배포 시 마주하는 다양한 상황을 학습하기 어렵다.
- **그래서 소수의 사람 데모를 재활용해 합성 데모를 대량 생성한다** — MimicGen류 방법은 사람 데모를 물체 중심 구간(subtask segment)으로 잘라, 새 장면의 물체 위치에 맞게 변형(transform)하고 이어붙여(stitch) 실행한 뒤, task에 성공한 것만 데이터셋에 남긴다(retain).
- **문제는 "성공한 것만 남기는" 마지막 단계다** — 널리 쓰이는 지표인 data generation rate(DGR, 생성 시도 중 성공해서 남는 비율)는 "얼마나 자주" 남는지만 잴 뿐, "무엇이" 남는지는 보지 못한다. 성공하기 쉬운 조건의 데모만 남는다면, 남은 데이터셋의 분포는 원래 탐색한 다양성을 보존하지 못한다.

### 지난 실험에서 관찰한 것 (4개 task)

지난 보고서에서 Square, Three Piece Assembly, Threading, Coffee 4개 task에 대해, 난이도 변형(D0 = 좁은 초기 배치 → D1 → D2 = 넓은 초기 배치)마다 생성을 500번씩 시도하고 성공·실패를 전부 기록해서 두 가지를 관찰했다.

- **관찰 1** — source 데모의 물체 배치와 합성 장면의 물체 배치가 멀수록(= 더 큰 transform이 필요할수록) 생성 성공률(DGR)이 낮아진다.
- **관찰 2 (Ancestry Bias)** — transform이 커질수록, 성공해서 남는 합성 데모가 소수의 "잘 옮겨지는" source 데모의 후손으로 쏠린다. MimicGen은 source의 궤적을 규칙 기반으로 변형해 쓰므로, 이 쏠림은 결국 행동 궤적(action trajectory)의 다양성 손실로 이어질 수 있다.

## 이번 실험의 질문

> 위의 두 관찰이 4개 task의 특수성인가, 아니면 MimicGen류 생성 일반의 성질인가?

이번 주에는 task를 9개로 늘려서 같은 실험을 반복했다. 핵심 메시지를 먼저 말하면, **task를 넓혀도 두 관찰이 전부 유지된다.** 9개 task 모두에서 초기 배치 영역을 넓힐수록 DGR이 떨어졌고(예외 0), 필터링 압력이 큰 task일수록 ancestry 쏠림이 심해졌다.

## 이번에 정비한 것 두 가지

task를 늘리기 전에, 지난 실험에서 걸렸던 문제 두 가지를 먼저 정리했다.

**첫째, "영역 확장"과 "영역 이동"을 분리했다.** MimicGen이 공개한 난이도 변형 중 일부는 영역을 넓히는 게 아니라 물체를 테이블 반대편으로 옮긴다(예: Threading D2는 needle과 tripod의 좌우를 뒤집는다). 이러면 "거리가 멀어져서 어려워진 것"과 "새로운 영역이라 어려워진 것"이 섞여 해석이 오염된다. 그래서 이런 task에는 **이전 단계 영역을 온전히 포함하면서 넓히기만 하는 확장 변형(E-시리즈)** 을 새로 정의했다. 넓힐 수 있는 한계는 감이 아니라 검증으로 정했다. 후보 영역의 극단 위치(모서리 4곳, 잡는 물체는 변 중앙 4곳 추가)에 물체를 고정하고 각 50번씩 생성을 시도해서, "그리퍼가 물체에 도달해 첫 subtask를 완수하는 비율"이 영역 안쪽과 비슷할 때만 그 경계를 확정했다. 이 관문에서 실제로 두 군데가 걸렸고(coffee는 두 물체 영역이 중앙에서 마주보는 안쪽 모서리에서 팔 진로가 막힘, stack은 로봇 팔 길이 한계인 원거리 모서리), 규칙대로 해당 경계를 2cm 줄여 통과시켰다.

**둘째, 생성 설정을 전 task 동일하게 통일했다.** source 데모 선택을 무작위로, 그리고 한 에피소드에는 source 데모 하나만 쓰도록 고정했다(일부 task의 공개 설정은 가까운 source를 골라주는 최근접 선택이라 시도 분포 자체가 원본 근처로 쏠린다). 이렇게 하면 에피소드마다 "어느 source에서 얼마나 떨어졌는지"가 하나의 값으로 정의된다. 참고로 이 변경의 영향은 Square에서 확인했는데, 무작위 선택 재실행의 DGR 21.0%는 MimicGen 논문의 같은 조건 실험값 22.4%와 잘 맞았다.

## Task & 변형 구성

| task | 하는 일 | 난이도 사다리 | 비고 |
|---|---|---|---|
| Square | 너트를 집어 페그에 끼움 | D0 → D1 → D2 | 공개 변형이 이미 확장형 |
| Threading | 바늘을 삼각대 구멍에 통과 | D0 → D1 → **D2E** | 공개 D2가 좌우 반전이라 확장판 신설 |
| Coffee | 커피 팟을 머신에 넣고 닫음 | D0 → **D1E** → **D2E** | 공개 D1·D2 모두 이동/반전이라 신설 |
| Three Piece Assembly | 베이스에 두 조각 조립 | D0 → D1 → D2 | 공개 변형이 확장형 |
| Stack | 큐브 2개 쌓기 | D0 → D1 → **D2E** | 공개는 D1까지라 한 단계 신설 |
| Stack Three | 큐브 3개 쌓기 | D0 → D1 → **D2E** | 상동 |
| Mug Cleanup | 머그를 서랍에 넣고 닫음 | D0 → **D1E** → **D2E** | 공개 D1이 머그 영역을 옆으로 밀어 신설 |
| Hammer Cleanup | 망치를 서랍에 넣고 닫음 | D0 → D1 | 공개 2단 사다리 |
| Coffee Preparation | 머그 배치→팟 꺼내 끼움 (5단계) | D0 → D1 | 공개 2단 사다리, long-horizon 대표 |

굵은 글씨가 이번에 새로 정의한 확장 변형이다. 모든 변형에서 시도 500번 고정, 성공·실패 전부 기록, source 데모는 각 task의 공개 10개를 그대로 썼다.

## 결과 1 — 영역을 넓힐수록 생성 성공률이 떨어진다 (9/9 task)

<img src="figures/motivation_ext/dgr_ladders_9tasks.png" width="100%">

- **9개 task 전부에서 단조 하락, 예외 없음** — 초기 배치 영역을 순수하게 넓히기만 해도(반대편 이동 없이) 생성 성공률이 한 단계도 오르는 일 없이 떨어진다. 지난 실험의 관찰 1이 task 일반의 성질임을 보여준다.
- **하락 폭은 task마다 다르다** — 쉬운 task는 완만하고(stack 70→59%), 정밀 삽입이나 긴 horizon이 낀 task는 가파르다(hammer cleanup 48→18%, mug cleanup 31→13%). 이 차이가 아래 결과 2의 열쇠가 된다.

## 결과 2 — 필터링 압력이 클수록 특정 source 데모로의 쏠림이 심해진다

성공한 데모만 남기는 필터링이 "어떤 source 데모의 후손이 살아남는가"를 얼마나 왜곡하는지를 ancestry skew로 쟀다. 시도 단계에서 상위 3개 source가 차지한 비중과, 살아남은 데모에서 상위 3개 source가 차지한 비중의 차이(%p)다. 시도는 무작위라 source별로 거의 균등한데, 살아남은 쪽이 쏠려 있다면 그 차이가 곧 필터링이 만든 편향이다.

<img src="figures/motivation_ext/ancestry_skew_dumbbell.png" width="100%">

- **영역을 넓히면 쏠림이 커진다** — 새로 추가한 task들에서 특히 뚜렷하다. coffee preparation은 +12.7 → +25.8%p, mug cleanup은 +9.6 → +21.4%p, hammer cleanup은 +0.5 → +15.2%p.
- **쉬운 task에서는 쏠림이 거의 없다** — stack(+4)과 coffee(+2)처럼 성공률이 높은 task는 어중간한 source도 살아남으므로 조상이 고르게 유지된다. 지난 실험에서 본 "필터링 압력이 커야 source 간 품질 차이가 드러난다"는 메커니즘 그대로다.

가장 심한 mug cleanup의 최광 변형(D2E)을 예로 보면, 시도는 source별로 42~60회로 고른데 살아남은 67개 중 절반 이상(55%)이 상위 3개 source에서 나왔고, **10개 중 2개 source는 후손을 거의 또는 전혀 남기지 못했다**(s7은 56회 시도에 생존 0). 유효 조상 수로 환산하면 10개 중 6.9개만 실질적으로 데이터에 기여한 셈이다(D0에서는 8.5개).

## 참고 — 공개 대칭이동 변형은 어떻게 확인됐나

지난 보고서에서 "일부 task는 분포가 대칭 변환이라 경향이 안 보인다"고 판단했는데, 이번에 그 판단을 정량으로 확인했다. 두 영역을 합친 박스로 거리를 다시 정규화해 보면, 대칭이동 변형의 배치는 확장 변형의 거리 범위와 **아예 겹치지 않는, 모든 source에서 순수하게 더 먼 구간**을 차지한다(threading: 확장 ≤0.37 vs 이동 ≥0.48). 확장 쪽의 거리-성공률 추세를 그 구간까지 연장해 보면 threading(위치만 반전)은 실측과 잘 맞고(예측 24.1% vs 실측 22.0%), coffee(머신의 놓인 방향까지 ~180° 반전)는 실측이 예측보다 27.9%p 낮다. 즉 위치 거리가 난이도의 1차 설명변수이되, 방향 반전 같은 자세 변화는 별도의 추가 난이도라는 부수 확인이다.

## Takeaway & Next Step

**Takeaway:**

- **관찰이 일반화됐다** — "영역을 넓힐수록 생성 성공률이 떨어진다"(9/9 task, 예외 0)와 "필터링 압력이 클수록 특정 source 데모의 후손으로 쏠린다"(최대 +25.8%p)가 task 4개 → 9개로 늘려도 유지된다. 4개 task의 우연이 아니라 성공-필터링 생성 방식 일반의 성질로 볼 근거가 생겼다.
- **쏠림의 크기가 처음으로 "0에 가까운 task"부터 "조상 1/3이 사라지는 task"까지 스펙트럼으로 잡혔다** — 필터링 압력(낮은 DGR)과 쏠림이 같이 움직이므로, 앞으로 어떤 task에서 이 편향을 걱정해야 하는지 예측할 단서가 된다.

**Next Step:** 그렇다면 이 편중이 최종 policy 성능을 실제로 깎는가? 이를 위한 정책 실험이 이미 돌아가는 중이다.

- 4개 task의 최광 변형에서 시도 6,250번짜리 큰 시도 풀을 만들고 있다 (성공·실패 전부 기록).
- 같은 풀에서 세 가지 500개 학습 데이터셋을 뽑는다 — ① 그냥 무작위 500(표준 파이프라인), ② source와의 거리 구간별 균등 500, ③ source 균형 500. 같은 풀에서 뽑으므로 차이는 오직 "어떤 500개를 남겼나"뿐이다.
- 세 데이터셋을 동일한 레시피(MimicGen 논문의 BC-RNN 설정 그대로)로 학습하고, 미리 고정해둔 200개 시작 상태에서 똑같이 평가해 짝지어 비교한다. 균등판이 원거리 시작 상태에서 이긴다면, 성공-필터링이 만든 편중이 policy 성능을 깎는다는 인과적 근거가 된다.
