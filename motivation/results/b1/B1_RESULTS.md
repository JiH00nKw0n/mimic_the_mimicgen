# B1 결과 — 도달성 프로브와 배치영역 동결 (2026-07-19 밤, aidas)

PLAN.md §1.5의 관문 실행 기록. 판정 원본은 `probe_report.json`, 런 데이터는 aidas `~/mimicgen_jihoonkwon/experiments/motivation_ic/b1_probe/`.

## 방법 요약

새로 정의한 확장 변형(D2E 계열)의 극단 위치들(모서리 4곳, 잡는 물체는 변 중앙 4곳 추가)에 물체를 작은 박스로 고정하고 위치당 50번 생성을 시도했다. 판정 지표는 task 성공이 아니라 **첫 subtask 완수율(그리퍼가 물체에 도달해 잡았는가)** — task 성공으로 판정하면 도달성과 transform 난이도가 섞이기 때문. 관문: 내부 기준 대비 −20%p 이내이고 절대 30% 이상.

## 과정에서 배운 것 (재현 시 주의)

1. mimicgen의 Stack 계열 env는 **모든 큐브의 배치 bounds가 동일해야** 생성자 assert를 통과한다 → 프로브는 큐브 전체를 한 박스에 고정 (박스 크기는 큐브 수 비례: 2개 10cm, 3개 15cm — 작으면 RandomizationError).
2. mimicgen `generate_dataset.py`는 내부 예외를 삼키고 종료 코드 0을 내보낸다 → 결과 파일(important_stats.json) 존재를 별도로 검사해야 가짜 성공을 막는다 (프로브 run 명령에 반영).
3. 미달 위치의 물리적 의미가 두 종류로 갈렸다: coffee는 **물체 간 간섭**(두 영역이 중앙에서 마주보는 안쪽 경계에서 팔 진로 차단), stack은 **순수 팔 길이 한계**(원거리 모서리). 논문 §1.5 서술에 이 구분을 쓸 것.
